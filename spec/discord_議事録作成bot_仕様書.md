# Discord 議事録作成 Bot 仕様書

## 1. システム概要
Discordのボイスチャンネルに常駐し、録音、Whisper APIによる文字起こし、ChatGPT APIによる議事録作成を自動で行うBot。

## 2. ディレクトリ構成と起動ショートカット
### 起動ショートカット
- プロジェクトフォルダ直下に `run_bot.bat`（Windows）または `run_bot.sh`（Mac/Linux）を配置
- 実行するだけで仮想環境の有効化とBotの起動が完了

### 設定ファイル
- `.env` : Discord / OpenAI APIキーの保管
- `config.md` : 議事録の要約プロンプトや出力形式の方針を記述（実行時に読み込み）

## 3. 機能詳細
### A. インタラクション（コマンド・参加・離脱）
- `/join` コマンド
  - ユーザーが現在参加している、または指定したボイスチャンネルにBotを参加させる
  - 参加と同時に録音準備状態へ移行

- 自動離脱機能
  - ボイスチャンネル内のBot以外のユーザー数を監視
  - ユーザー数が0人になった瞬間に自動離脱
  - 離脱をトリガーに音声データ処理を開始

### B. 音声処理・テキスト化（Whisper API）
- 音声保存
  - 録音データを一時的なWAVまたはMP3ファイルとして保存

- 文字起こし
  - Whisper API（`whisper-1`）を使用
  - タイムスタンプ付きプレーンテキストを生成

### C. AI要約（ChatGPT API）
- 議事録生成
  - Whisperの出力テキストをChatGPTへ送信
  - `config.md` に記載された作成方針をシステムプロンプトとして使用

- 出力方法
  - Discordの指定テキストチャンネルへ投稿
  - またはMarkdownファイルとして保存

## 4. 議事録作成方針（config.md）の構成案
- フォーマット: 決定事項 / ネクストアクション / 議論の背景 の3段構成
- トーン: 丁寧なビジネス口調
- 重点: 技術的な仕様は詳細に、雑談はカット

## 5. アーキテクチャ概要
1. Botがボイスチャンネルに参加
2. 音声録音
3. チャンネル無人化を検知し自動離脱
4. Whisper APIで文字起こし
5. ChatGPT APIで議事録要約
6. 結果を出力

## 6. 開発に向けた Claude へのプロンプト案
```
Discordの議事録作成Botを作成してください。
言語はPython、ライブラリは discord.py（または disnake / nextcord）を想定しています。

既存の録音ロジックを組み込んでください。

/ join コマンドでVCに参加し、VCが空になったら自動離脱する機能を実装してください。

離脱後、OpenAIのWhisper APIで文字起こしし、その結果を config.md に記載した指示に従って
ChatGPT APIで要約するフローを実装してください。

プロジェクト直下でBotを起動するための run_bot.bat の中身も作成してください。
```
